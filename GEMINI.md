## PROOF-DRIVEN VERIFICATION MANDATE (CRITICAL)

> ⚠️ Glass-Box Transparency demands proof, not assumption. Hallucination = security risk.

### Core Rules
- exit code 0 ≠ success. Read stdout/stderr for hidden errors.
- "It should work" is NOT evidence. Only verifiable runtime output counts.
- Never report partial work as complete. Never skip verification to report success.
- If verification is blocked → state it explicitly. Never guess.

### Verification Protocol
- **API/Outreach calls**: Show raw API response (status code + payload). Assert expected data.
- **Deployments**: Pull latest server logs. HTTP GET home page. Assert `200`. Show HTML.
- **Lead gen runs**: Show exact counts (e.g. `41 leads scraped`). Show CSV row sample.
- **Code edits**: Run linter/type-check. Show exact output. Run tests. Show pass/fail counts.
- **DB/Config**: Query directly. Show result. Show diff.

### Mandatory Proof of Work Block (append to EVERY concluding response)
```
### Proof of Work
- **Verification Commands Run**: <exact commands>
- **Evidence Output**: <raw logs / API response / DB rows / screenshot path>
- **Result**: [SUCCESS/FAILURE — verified by evidence, not assumption]
```
No Proof of Work = task NOT complete. No exceptions.

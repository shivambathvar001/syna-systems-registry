# Syna Systems Ideal Customer Profile (ICP) Framework

## Target: The "Operationally Strained" Mid-Market (10-50 Employees)

### 1. The Core Signal: "Hiring for Friction"
We target companies hiring for roles that are essentially "Human Middleware":
- **Operations Coordinators:** Moving data between systems.
- **Process Analysts:** Documenting manual steps.
- **SDRs/BDRs:** Doing manual lead research.
- **Customer Support:** Handling repetitive, high-volume queries.

### 2. Industry-Agnostic Pain Points
- **Manual Data Re-entry:** Copying data from Email -> CRM -> Sheets.
- **Approval Bottlenecks:** Waiting for Slack/Email responses to move to the next step.
- **Lack of Visibility:** Leadership doesn't know where a specific task is in the pipeline.
- **Scaling Pain:** "To grow 2x, we need to hire 2x more people." (Linear scaling).

### 3. Selection Criteria (The 50-Lead Sprint)
| Signal | Why it Matters | How to Find |
| :--- | :--- | :--- |
| **Hiring Ops/Admin** | High manual workload. | LinkedIn Jobs / Indeed. |
| **Tech Stack Complexity** | Uses 5+ SaaS tools (HubSpot, Slack, Zendesk, etc.). | BuiltWith / Wappalyzer. |
| **High Document Volume** | Legal, Logistics, Recruitment, Real Estate. | Industry filter. |
| **Recent Funding/Growth** | Urgency to scale without bloating headcount. | Crunchbase / News. |

## The "Immediate Yes" Value Proposition
"We don't build chatbots. We build **Operational Infrastructure** that turns manual bottlenecks into observable, automated workflows. You own the code; we eliminate the friction."
